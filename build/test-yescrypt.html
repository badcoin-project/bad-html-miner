yescrypt-WASM Test Page

Verifies that the Emscripten build of BadCoin's yescrypt source loads in a browser and produces hashes. Use this as the smoke test after running build-yescrypt-wasm.sh.

How to read the result

  • Rate > 5 H/s and a stable hash hex: WASM is working. You can move to Turn 3 of the build plan (wire WASM into the miner).
  • Rate < 1 H/s: WASM loaded but is doing something pathological. Check browser DevTools console for errors. Most likely cause: the module is being recompiled on every hash.
  • "Failed to load WASM" or other init error: Browser cannot load the WASM. Serve via python3 -m http.server rather than file://.
  • Hash hex is all zeros: WASM is being called but input/output is broken. Check yescrypt-wrapper.js for cwrap signature.

What this test does

Loads yescrypt-inline.js (preferred, single file) or yescrypt.js+yescrypt.wasm (split). Initializes the module once. Allocates an 80-byte input buffer and a 32-byte output buffer in WASM memory. Hashes a varied input N times. Measures wall-clock rate. Displays the last hash hex.
